# sensitive-words — 敏感词检测参考

> 本模块由 ink.skill 在审计章节时自动参考（作为 reference/audit.md 的补充）。
> 非 LLM 规则检测：确定性地扫正文是否含敏感词，命中按 severity 上报。
> 不需要 LLM 判断，属于机械规则层（同 meta-leaks、机械规则检查）。

---

## 自定义词表

作者可在 `book_rules.md` 里添加 `customSensitiveWords` 字段（列表），LLM 扫描时一并检测，全部按 severity=warning 处理：

```yaml
customSensitiveWords:
  - "某词A"
  - "某词B"
```

---

## 审计集成

每次 Step 7 审计时，LLM 对照本文件三个词表扫描正文：

1. 发现 block 级（政治敏感词）→ 上报 `severity: critical`，category: `敏感词`
2. 发现 warn 级（色情/极暴力/自定义）→ 上报 `severity: warning`，category: `敏感词`
3. 报告格式：`检测到<类别>：「词A」×N 次、「词B」×M 次`

审计维度 27（敏感词检查）= 本文件定义的检测逻辑。

---

## 说明

- 本模块是**规则性检测**，不需要 LLM 语义判断
- 扫描是逐词精确匹配（不是语义近似匹配）
- 同一词多次出现：计数后一次性上报（不是每次出现报一条 issue）
- 书名/地名引用了敏感词（如书中书的书名用书名号包裹）：LLM 可酌情忽略，但须在 info 级注明

## 🚨 禁止清单外的"自我审核"替换（v0.1.13 硬律）

**本模块定义的敏感词清单是唯一的执法范围**。清单包含：

- **block（政治敏感词）**：18 个，见上方 POLITICAL_WORDS
- **warn（色情敏感词）**：16 个，见上方 SEXUAL_WORDS
- **warn（极端暴力词）**：10 个，见上方 VIOLENCE_EXTREME
- **warn（自定义词）**：作者在 `book_rules.md` 的 `customSensitiveWords` 字段列出的词

**清单外的任何词，LLM 不得以"平台审核"/"避免敏感"/"替换更稳"之类的理由自作主张替换**，包括但不限于：

- 武器类：**枪**、**刀**、**剑**、**火药**、**炸弹**（只要在书里出场的武器都要保留原词，作者写"枪"就是"枪"，不能改"铁管"/"长棍"/"冷兵"）
- 成人题材但非清单词：**酒**、**烟**、**血**、**裸体**（非清单就不改）
- 现代化场景词：**手机**、**电脑**、**警察**、**医院**（属作者意图的场景设置，不改）
- 方言 / 俚语 / 粗口但非清单词：**妈的**、**见鬼**、**王八蛋**（作者风格，不改）

**为什么要这条硬律**：LLM 训练数据里充斥"中文网文平台对现代武器/成人元素敏感"的说法，Writer / Reviser 看到"平台写作"这种触发词会自动启动**过度防御模式**，把自己幻觉出的"可能被审核的词"静默替换掉，作者往往要对比正文才发现。本模块的 3 类词表 + 作者自定义词表是 LLM 唯一合法的审核对象，**清单外一律按作者原意保留**。

**违反时的后果**：作者读正文发现被静默改词（枪→铁管、酒→茶、血→汁等）→ 可要求 LLM 回退修订、重审、记入 PROGRESS.md 操作时间线作为 drift 标记。

**自检触发点**：Writer / Reviser 准备把某个词替换成另一个词之前 → 问自己"这个词在 sensitive-words 4 个清单里吗？"——不在 → **不换**。

（根因详见项目 `CLAUDE.md` §9 待办里 2026-04-16 的 "LLM 静默过度审核" bug 记录。）
