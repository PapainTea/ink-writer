# post-write-validator — 写后机械规则校验

> 本模块由 ink.skill 在**写章 pipeline Step 6** 自动调用（不需要作者显式触发）。
> 用途：在 Writer 产出正文之后、审计（Step 7）之前，跑 12 条**确定性机械规则**扫描正文，抓最常见的硬性违规。
> 机制：全部规则可用正则 / grep 直接匹配，**不需要 LLM 语义判断**。
> 和 `reference/audit.md` 的区别：audit 是 37 维**语义**审查（要 LLM 判断），post-write 是**机械**扫描（确定性）；两者并行，前者找"不合理"，后者找"硬违规"。
> 相关：`reference/writing-bans.md`（同一套禁令规则的**独立可读副本**，Writer / Reviser / Reanalyze 都应 Read）、`scripts/verify-chapter.py`（Step 12 最后兜底同一批规则）。

---

## 用途

写章后机械规则检查，**零 LLM 成本**。捕捉仅靠 prompt 约束无法保证的违规——确定性规则在每次章节生成后自动运行，catch prompt-only rules cannot guarantee 的违规项。

每条违规包含：`rule`（规则名）/ `severity`（`error` 或 `warning`）/ `description`（描述）/ `suggestion`（修改建议）。

## 12 条检查规则

### 1. 禁止"不是…而是…"句式 `[error]`

- **正则**：`/不是[^，。！？\n]{0,30}[，,]?\s*而是/`
- **描述**：出现了"不是……而是……"句式
- **建议**：改用直述句

### 2. 禁止破折号 `——` `[error]`

- **检测**：`content.includes("——")`
- **描述**：出现了破折号"——"
- **建议**：用逗号或句号断句

### 3. 转折/惊讶标记词密度 `[warning]`

- **标记词**：`仿佛` / `忽然` / `竟然` / `猛地` / `猛然` / `不禁` / `宛如`
- **阈值**：`≤ 1 次 / 3000 字`（即 `markerLimit = max(1, floor(content.length / 3000))`）
- **描述**：报告总计数、上限、各词明细
- **建议**：改用具体动作或感官描写传递突然性

### 4. 高疲劳词 `[warning]`

- **来源**：`bookRules.fatigueWordsOverride`（若提供）否则 `genreProfile.fatigueWords`
- **阈值**：每词 `≤ 1 次 / 章`
- **建议**：替换多余的该词为同义但不同形式的表达

### 5. 元叙事 / 编剧旁白 `[warning]`

- **正则 patterns**：
  - `/到这里[，,]?算是/`
  - `/接下来[，,]?(?:就是|将会|即将)/`
  - `/(?:后面|之后)[，,]?(?:会|将|还会)/`
  - `/(?:故事|剧情)(?:发展)?到了/`
  - `/读者[，,]?(?:可能|应该|也许)/`
  - `/我们[，,]?(?:可以|不妨|来看)/`
- **触发**：命中任一即报一次（break 后不重复）
- **建议**：删除元叙事，让剧情自然展开

### 6. 分析报告式术语 `[error]`

- **词表 `REPORT_TERMS`**：`核心动机` / `信息边界` / `信息落差` / `核心风险` / `利益最大化` / `当前处境` / `行为约束` / `性格过滤` / `情绪外化` / `锚定效应` / `沉没成本` / `认知共鸣`
- **描述**：正文中出现分析报告术语：……
- **建议**：这些术语只能用于 `PRE_WRITE_CHECK` 内部推理，正文中用口语化表达替代

### 7. 作者说教词 `[warning]`

- **词表 `SERMON_WORDS`**：`显然` / `毋庸置疑` / `不言而喻` / `众所周知` / `不难看出`
- **建议**：删除说教词，让读者自己从情节中判断

### 8. 全场震惊类集体反应 `[warning]`

- **正则 patterns**：
  - `/(?:全场|众人|所有人|在场的人)[，,]?(?:都|全|齐齐|纷纷)?(?:震惊|惊呆|倒吸凉气|目瞪口呆|哗然|惊呼)/`
  - `/(?:全场|一片)[，,]?(?:寂静|哗然|沸腾|震动)/`
- **触发**：命中任一即报一次
- **建议**：改写成 1-2 个具体角色的身体反应

### 9. 连续"了"字 `[warning]`

- **切句**：按 `。！？` 切分，过滤长度 > 2 的句子
- **阈值**：最大连续包含"了"的句数 `≥ 6`
- **建议**：保留最有力的一个"了"，其余改为无"了"句式

### 10. 段落过长 `[warning]`

- **切段**：按空行切分
- **阈值**：`≥ 2` 个段落字符长度 > 300
- **建议**：长段落拆分为 3-5 行的短段落，在动作切换或情绪节点处断开（手机阅读适配：50-250 字/段为宜）

### 11. 本书禁忌 `[error]`

- **来源**：`bookRules.prohibitions`
- **匹配**：长度 `2 ≤ len ≤ 30` 的条目走精确子串匹配；长度 > 30 的条目视为概念性规则，跳过机械检查（仅在 prompt 层约束）
- **建议**：删除或改写该内容

### 12. 禁止 Markdown 结构化标记泄漏到正文 `[error]`

LLM 写作时容易**把自己常用的 markdown 语法当作"场景分隔符""强调符"带进正文**，这些是给 Claude Code 对话用的格式，不是小说叙事的正常语言。一经发布到起点/番茄等平台会显得业余。

- **禁止项**：
  - 水平分隔线 `---`（独占一行）—— 典型错误：用 `---` 分场景
  - 三级及以下标题 `###` / `####`—— 小说正文只允许首行 `# 第 N 章 <标题>`，不允许章内再有 `#` 标题
  - 粗体 `**文字**` / 斜体 `*文字*`—— 小说用词语本身表达强调，不用符号
  - 无序列表 `- ` / `* `（行首）—— 小说不用列表结构
  - 有序列表 `1. ` / `2. `（行首）—— 同上
  - 行内代码 `` `文字` `` / 代码块 ` ``` `—— 小说不含代码
  - Markdown callout `> [!note]` 之类—— 只用于审计 md，不用于正文

- **检测**：
  - 空行之间独占一行的 `---` → 命中
  - `^#{1,6} ` 开头（除第一行章节标题外）→ 命中
  - 除去引号内对话，出现 `**...**` 包裹的文本 → 命中
  - 行首 `- ` / `* ` / `1. ` 等列表标记 → 命中（正文第一个字符几乎不会以这些开头）
  - ` ``` ` 或行内 `` ` `` → 命中

- **建议**：
  - 场景切换用空行 + 场景首句重新建立时地人物，不用 `---`
  - 强调用词本身（"他真的做了"而不是"他**真的**做了"）
  - 列举用自然语言（"有三件事：第一……第二……第三……"而不是 `1. / 2. / 3.`）
  - 章节标题永远只在第 1 行 `# 第 N 章 <标题>`，之后整章不出现任何 `#`

- **触发**：一经命中即 error，走 spot-fix 自动清理

## 执行流程

1. 写章节生成完成后，自动运行 `validatePostWrite(content, genreProfile, bookRules)`
2. 收集所有违规项
3. **处理策略**：
   - `severity: error` → 自动触发 spot-fix 修订
   - `severity: warning` → 报告给用户，不自动修
4. **防无限循环**：同一个问题自动 spot-fix 2 次后仍未通过 → 停止自动修复，交给用户人工处理

## 跨章重复检查（附加）

`detectCrossChapterRepetition` 从当前章提取 6 字中文 ngram，若某 ngram 在当前章出现 `≥ 2` 次**且**在近期章节正文中也出现过，则累计；累计 `≥ 3` 条时报 `跨章重复 [warning]`：

- **建议**：变换动作描写和场景用语，避免跨章节机械重复
