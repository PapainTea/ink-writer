# fanfic — 同人小说写作专用参考

> 本模块由 ink.skill 在以下条件时 Read：
> - 新建书时体裁含"同人" / "fanfic" / 二创
> - 书目录下存在 `story/fanfic_canon.md`（同人正典参照）
> - 用户消息含"同人"、"二创"、"fanfic"
>
> 同人写作在通用写作规则基础上额外遵守原作 canon，不可 OOC（角色性格出格）、不可违反原作世界规则。

---

## 触发条件

| 条件 | 触发行为 |
|------|---------|
| 新建书时体裁为同人 | 进入 fanfic-canon-importer 流程（见下方子功能 1）|
| 书根存在 `story/fanfic_canon.md` | 写章/审计时自动加载本模块 |
| 用户说"同人" / "二创" / "fanfic" | 手动触发本模块 |

---

## 同人模式（fanficMode）

| 模式 | 说明 | 典型场景 |
|------|------|---------|
| `canon`（原作向）| 严格遵守原作设定，填充空白角度 | 补全原作未详写的场景 |
| `au`（平行世界）| 世界规则可改，角色核心性格保留 | 现代 AU / 古代 AU |
| `ooc`（角色性格偏离）| 角色在极端情境下可偏离性格底色 | 极端情境/黑化 |
| `cp`（配对中心）| 以角色互动和关系发展为核心 | 配对向同人 |

---

## 子功能 1: fanfic-canon-importer（原典导入）

**用途**：用户提供原作素材（原作 wiki / 章节正文 / 设定集），导入生成 `story/fanfic_canon.md`。

**触发**："帮我导入原作" / "新建同人书" / "建立正典参照"

### 导入 System Prompt

```
你是一个专业的同人创作素材分析师。你的任务是从用户提供的原作素材中提取结构化正典信息，供同人写作系统使用。

同人模式：<modeLabel>（原作向/AU/OOC/CP 之一）

你需要从原作素材中提取以下内容，每个部分用 === SECTION: <name> === 分隔：

=== SECTION: world_rules ===
世界规则（地理、物理法则、魔法/力量体系、阵营组织、社会结构）。
如果原作素材不包含明确的世界规则，从已有信息合理推断。

=== SECTION: character_profiles ===
角色档案表格，每个重要角色一行：

| 角色 | 身份 | 性格底色 | 语癖/口头禅 | 说话风格 | 行为模式 | 关键关系 | 信息边界 |
|------|------|----------|-------------|----------|----------|----------|----------|

要求：
- 语癖/口头禅必须从原文中精确提取，如有的话
- 说话风格描述该角色的语气、用词偏好、句式特征
- 行为模式描述该角色在特定情境下的典型反应
- 信息边界标注该角色知道什么、不知道什么
- 至少提取 3 个角色，不超过 15 个

=== SECTION: key_events ===
关键事件时间线：

| 序号 | 事件 | 涉及角色 | 对同人写作的约束 |
|------|------|----------|------------------|

按时间/出现顺序排列，标注每个事件对同人创作的约束程度。

=== SECTION: power_system ===
力量/能力体系（如果适用）。包括等级划分、核心规则、已知限制。
如果原作没有明确的力量体系，输出"（原作无明确力量体系）"。

提取原则：
- 忠实于原作素材，不捏造原作中没有的信息
- 信息不足时标注"（素材未提及）"而非编造
- 角色语癖是最重要的字段——同人读者最在意角色"像不像"
```

### 导入 User Prompt

```
以下是原作《<书名>》的素材：

<原作素材内容（超过 5 万字时截断前 5 万字）>
```

```markdown
# 同人正典（《<原作书名>》）

## 世界规则
<world_rules 段提取内容>

|------|...

## 关键事件时间线
| 序号 | 事件 | 涉及角色 | 对同人写作的约束 |
|------|...

<power_system 段提取内容>
```

## 子功能 2: fanfic-prompt-sections（同人写作 prompt 增量）

**用途**：写同人时，在 reference/write.md 的通用写作规则基础上追加同人专用约束。

### 按 fanficMode 追加的写作前置语

**canon（原作向）**：
```
你正在写原作向同人。严格遵守正典：
- 角色的语癖、说话风格、行为模式必须与原作一致
- 世界规则不可违反
- 关键事件时间线不可矛盾
- 可以填充原作空白、探索未详述的角度
```

**au（平行世界）**：
```
你正在写 AU（平行世界）同人：
- 世界规则可以改变（已在 allowedDeviations 中声明的偏离）
- 角色的核心性格和说话方式应保持辨识度——读者要能认出是谁
- AU 设定偏离必须内部一致（改了一条规则，相关的都要跟着变）
```

**ooc**：
```
你正在写 OOC 同人：
- 角色在极端情境下可以偏离性格底色
- 但偏离必须有情境驱动，不能无缘无故变性格
- 保留角色的语癖和说话特征——即使性格变了，说话方式也应有辨识度
```

**cp**：
```
你正在写 CP 同人，以角色互动和关系发展为核心：
- 配对双方每章必须有有效互动
- 互动风格要有化学反应——不是两个人在同一个场景各干各的
- 关系发展应有节奏感：推进、试探、阻碍、突破
```

### 写作前自检追加（在 PRE_WRITE_CHECK 末尾追加）

**canon**：
```
- 正典合规检查：本章是否违反原作设定？角色对话是否符合原作语癖？
- 信息边界检查：角色是否引用了不该知道的信息？
```

**au**：
```
- AU 偏离清单：本章改变了哪些世界规则？改变是否内部一致？
- 角色辨识度检查：读者能否从对话中认出角色？
```

**ooc**：
```
- OOC 偏离记录：角色在哪些方面偏离了性格底色？偏离驱动力是什么？
- 语癖保留检查：即使 OOC，说话方式是否还有原作特征？
```

**cp**：
```
- CP 互动检查：配对双方本章是否有有效互动？关系发展是否推进？
- 互动质量检查：互动是否有化学反应（不是各干各的）？
```

允许的偏离（不视为违规，作者在 book_rules.md 里声明 `allowedDeviations`）：

```
允许的偏离：
- <偏离项A>
- <偏离项B>
```

### 角色语音参照段（写章时注入 context）

从 `fanfic_canon.md` 的角色档案表提取各角色语癖/说话风格，在 user prompt 里添加：

```markdown
## 角色语音参照（同人写作专用）

以下角色的对话和行为必须参照原作特征。写对话时，先想"这个角色在原作里会怎么说"。

### <角色A>
- 口头禅/语癖：<原作提取>
- 说话风格：<原作提取>
- 典型行为：<原作提取>

### <角色B>
- ...
```

### 正典参照段（写章时注入 context）

在 user prompt 末尾追加：

```markdown
## 同人正典参照

<按 fanficMode 的前置语（见上方）>

以下是原作正典信息，写作时必须参照：

<fanfic_canon.md 完整内容>
```

## 审计集成

reference/audit.md 的维度 34-37 是同人专用：

| 维度 | 说明 | 触发条件 |
|------|------|---------|
| 34. 角色还原度 | 对话语癖、说话方式、行为是否符合 fanfic_canon.md 角色档案 | `fanfic_canon.md` 存在 |
| 35. 世界规则遵守 | 是否违反 fanfic_canon.md 中的世界规则（地理/力量体系/阵营）| 同上 |
| 36. 关系动态 | 关系推进是否合理、是否与 fanfic_canon.md 中的关键关系对齐 | 同上 |
| 37. 正典事件一致性 | 是否与 fanfic_canon.md 中的关键事件时间线矛盾 | 同上 |

当 `story/fanfic_canon.md` 存在时，这 4 个维度自动启用（不需要在 `book_rules.md` 里额外配置）。

OOC 模式下维度 34（角色还原度）降级为 info（仅记录偏离，不判定失败）。

## 与通用 genre 的关系

同人书通常选体裁为"其他/通用"（other），在此基础上额外加载本模块。也可以组合：如"同人+仙侠"，则同时加载 genres/xianxia.md 和本模块（两套规则叠加，以本模块为优先）。
