## Genre Prohibitions

- System rules changing arbitrarily after being established — readers track every number
- Unexplained power jumps — growth must follow established system logic
- "Blue Box Madness" — stat dumps every chapter or pages-long stat sheets
- Instant mastery — acquiring a skill and immediately excelling at it
- Overpowered MC from the start — removes all tension
- System message overload interrupting action scenes
- Female characters reduced to "perfect girlfriend" or "sickly daughter" tropes

## System Design Rules

- Once stats, skills, and system rules are established, they cannot be contradicted
- Stat blocks punctuate achievement moments, not routine actions
- Derivative stats (HP, Mana) must depend logically on primary stats
- Skill unlocks must feel earned — tied to risk, sacrifice, or problem-solving
- Same-type resource absorption must show diminishing returns, not flat gains
- System UI (blue boxes) appears after action/revelation, never mid-combat
- Give the system a consistent voice/personality — formal, archaic, playful, or clinical

## Pacing Guidance

- Chapter length sweet spot: 2.8k-3.5k words depending on stat density
- Chapter structure: Hook/recap -> Action/exploration -> System interaction/stat gain -> Cliffhanger
- Early chapters: frequent level-ups to hook readers (every 1-3 chapters)
- Mid-story: harder gains, every 5-10 chapters; narrative tension rises
- Late story: tier/rank transitions are rare and climactic
- Test pacing: Book 3 MC decisively defeats Book 1 version, but challenges never feel trivial
- Describe stats in narration first (audiobook-friendly), then include stat sheet for detail readers

## Supporting Character Independence Standard

Applies to **all** recurring non-MC characters regardless of gender. Each supporting character with more than cameo screen time must satisfy:

- **Independent goal**: A stated or demonstrated objective that is not "support the MC" — own quest, debt, grudge, ambition, or survival target
- **Independent competence**: At least one domain where they outperform the MC at time of introduction (tactics, crafting, social, lore, a specific skill tree branch)
- **Independent stat trajectory**: Their level / skills / tier advance on a visible schedule even in chapters they don't headline; they are not frozen props waiting for MC scenes
- **Refusal capacity**: Shown or implied willingness to say no to the MC at non-trivial cost at least once per arc
- Avoid reducing any character to a single-trope slot (perfect girlfriend / sickly daughter / loyal retainer / scheming rival). Each tropey template must be broken by at least one action that contradicts the trope before the arc closes

## Stat UI Budget (per-scene and per-chapter caps)

Numbers in LitRPG are the genre's signature, but unbudgeted blue boxes wreck pacing. Default budget (may be overridden by book_rules):

- **Per-scene stat display**: ≤ 120 words of raw system UI (blue box / table / log block) per scene; if exceeded, cut into narration or defer to end-of-chapter summary
- **Per-chapter stat refresh frequency**: at most 2 full stat-sheet appearances per chapter — typically one mid-point (earned moment) and one end-of-chapter (checkpoint), never during active combat ticks
- **System message interrupts in combat**: ≤ 1 short message (system voice, ≤ 20 words) per combat beat, grouped into a batch-notification at beat's end when possible
- **Level-up / tier-up moments**: allowed a single longer UI block (up to 250 words) but narration must carry emotional weight before the box fires
- **Loot table dumps**: itemize only what the MC examines / equips / reacts to; remaining loot summarized in one line

## LitRPG Checklist

After writing, verify:

- [ ] Did each supporting character on-page this chapter show at least one independence marker (goal / competence / stat progress / refusal)?
- [ ] Is the total raw UI word count within the per-chapter budget (≤ 2 full sheets + combat micro-notifications)?
- [ ] Were all numerical gains tied to a logged cause (kill credit, skill use count, quest reward), not handwaved?
- [ ] Did system UI appear AFTER an action/revelation beat, not mid-swing?
- [ ] If a new skill was unlocked, was risk / sacrifice / problem-solving visible on-page?
